var 똑같은 이름으로 재선언하는것이 가능하다.
똑같은 이름으로 변수가 만들어지면 덮어쓰가 된다.
변수에 값을 계속 대입할 수 있다.

let 똑같은 이름의 변수를 또 만들 수 없다.
변수에 값을 계속 대입할 수 있다.

const 똑같은 이름의 변수를 만들 수 없다.
선언과 동시에 초기화를 시켜야 한다.
다른값을 대입하는것이 불가능하다.

자료형(data type)
현실세계에서 사용하던 데이터를 디지털화 시켜서 사용하려고 할 때 지정하는 타입

변수명을 짓는 규칙
1. 알파벳과 숫자는 쓸 수 있다.(단, 숫자가 첫글자로 올 수 없다)
2. 밑줄( _) 과 달러기호 $를 빼고는 쓸 수 없다.

let name;
let user1
let _private
let $value
let 이름;
let 1stUser -> X (숫자가 첫글자로 올 수 없다)
let user name -> 띄어쓰기 안된다.

자바스크립트에서 이미 의미가 정해진 키워드는 변수명으로 쓸 수 없다.
let, var, const, if, else, for, while,return...

대소문자 구분
완전히 다른 변수로 구분한다.

let user = 'tom';
let User = 'Jane';


권장되는 네이밍 관례
카멜(낙타)표현식
camelCase
단어가 여러개일 때 첫단어는 소문자, 이후 단어의 첫글자는 대문자

let userName
let totalPrice
let maxCount
let createdAt

상수는 이름을 대문자와 언더스코어로 짓는다.
const MAX_RETRY = 5;
const PI = 3.141592;

역할이 예상 되도록 짓는것이 좋다.
let x -> X
let userAge -> O

자료형/역할에 따라 네이밍 패턴을 정하면 좋다.
boolen타입의 변수 : is, has, can should 등으로 시작하면 좋다.

let isLoggedIn = true;
let hasPermission = false;
let canEdit = true;
let shouldReload = false;
