React
meta에서 개발한 자바스크립트 라이브러리

html,css,js로만 개발을 했을 때는 하나의 화면에 다 작성을 했어야 했다.

React는 화면을 잘게 쪼개서 "컴포넌트" 단위로 사용

컴포넌트
잘게 쪼갠 UI의 단위

함수형 컴포넌트

const 함수명 = () => {

    일반 Js 코드를 작성할 수 있다.

    return(UI(html) 작성)
}

JSX(JavaScript XML)문법
JS안에서 HTML처럼 작성하는 문법
브라우저는 JSX를 직접 실행 X
빌드도구(Babel)가 JSX를 JS코드로 변환

const hello = <h1>Hello</h1>

return(
    <div>
        내용
    </div>
)

const name = "kim";

return(
    <div>
	<h1>이름 : {name}</h1>
    </div>
)

npm run dev
↓
package.json의 "dev":"vite"가 실행이 된다.
↓
vite 개발 서버가 뜨게 되고, 브라우저 주소를 반환받는다.
(localhost:5173)
↓
브라우저는 index.html을 먼저 받는다.
↓
Main.jsx에서
document.getElementById('root')를 통해서 <div>를 찾는다.
createRoot()를 통해 <div>를 React가 관리하는 객체로 만든다.
render()메서드를 통해서 App 컴포넌트를 브라우저에 출력한다.
↓
App 컴포넌트가 반환한 JSX안에 다른 컴포넌트가 있다면, React는 그 컴포넌트들도 계속 호출을 한다.
↓
컴포넌트 트리를 따라서 내려가면서 실행
↓
Virtual DOM에서 비교 후, 실제 DOM을 업데이트한다.


<StrictMode> : 코드를 테스트
선언되어 있으면 개발모드에서 일부 로직이 한 번 더 실행되는 것 처럼 보일 수 있다.
